# SQL E-commerce Sales & Customer Analytics
Advanced SQL data analytics project evaluating financial performance, customer spending habits, and channel efficiency for an e-commerce platform using PostgreSQL.

## 📌 Project Overview
A data-driven analytics project dedicated to evaluating the financial and operational performance of an e-commerce business. The analysis processes transactional data across multiple store formats to extract behavioral trends, measure store efficiency, and identify high-value customer cohorts.

---

## 🛠️ Tools & Technologies Used
* **Environment:** PostgreSQL / DBeaver
* **Advanced SQL Concepts:** Multi-table JOINS, CTEs, Window Functions (Ranking, Running Totals), Subqueries, Aggregate Functions.

---

## 📈 Key Metrics Calculated
* **Financial Health:** Total Revenue, Order Volume, and Monthly Growth.
* **Customer Value:** Average Order Value (AOV) and Frequency of Purchases.
* **Segmentation:** Top performing items and high-spending customer tiers.

---

## 📊 Key Findings & Business Insights
* **Channel Efficiency:** While offline brick-and-mortar stores generated a lower total volume of items sold, they demonstrated a significantly **higher Average Order Value (AOV)** compared to online storefronts.
* **Customer Spending:** Identified the core high-spending customer segment responsible for the majority of luxury tier item purchases.
* **Trends:** Uncovered month-over-month revenue shifts, allowing stakeholders to align inventory preparation with historical consumer purchasing spikes.
